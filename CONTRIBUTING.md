# Contributing to StealthFlashlight

First off, thank you for considering contributing to StealthFlashlight. It's people like you that make StealthFlashlight such a great tool.

## How to Contribute

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally.
3. **Create a new branch** for your feature or bug fix.
4. **Commit your changes** with descriptive commit messages.
5. **Push your branch** to your fork.
6. **Submit a Pull Request** to the `master` branch of the main repository.

## Development Setup
Please refer to the [README.md](README.md) for instructions on setting up your local development environment, including how to configure Firebase for your local build.

## Code Style
*   We use standard `flutter analyze` linting rules. Please run `flutter analyze` before committing your code.
*   Ensure your code is properly formatted using `flutter format .`.

## Reporting Bugs
If you find a bug, please use the Bug Report issue template to report it. Provide as much detail as possible, including steps to reproduce the issue.

## Suggesting Features
If you have an idea for a new feature, please use the Feature Request issue template. Describe the feature and why it would be beneficial.
